# 《选择之间》内容审稿器 v0.1

## 一、审稿目标

审查一篇《选择之间》内容是否只是“结构完整”，还是确实具备：

- 命题锋利度
- A/B 两难张力
- 价值冲突清晰度
- 分镜画面感
- 旁白传播力
- 平台标题吸引力
- 是否存在模板化、空泛化、错配主题

## 二、审稿对象

默认审查：

`outputs/*.md`

也允许用户指定某个文件，例如：

`outputs/growth-rules-freedom.md`

## 三、评分维度

每篇内容按 100 分审查：

### 1. 命题锋利度，20 分

检查标题和命题是否能让人立刻产生停顿、争议或代入感。

- 低分表现：太平、太像作文题、没有冲突。
- 高分表现：一句话就能让人想选边、想反驳、想评论。

### 2. 核心价值冲突，20 分

检查全文是否始终围绕一个清晰冲突展开。

- 低分表现：努力、自由、成长、规则混在一起但没有主轴。
- 高分表现：例如“努力 vs 驯化 / 适应规则 vs 成为自己”贯穿全文。

### 3. A/B 两难张力，15 分

检查 A/B 是否真的都有道理。

- 低分表现：一边明显正确，一边像靶子。
- 高分表现：两边都有代价，读者很难轻易选。

### 4. 内容原创感，15 分

检查是否有模板味、套话、空话。

- 低分表现：大量“不是……而是……”但没有具体内容。
- 高分表现：有具体判断、具体语气、具体洞察。

### 5. 分镜画面感，15 分

检查 6 个分镜是否可拍、可画、可用于 AI 生成画面。

- 低分表现：只有抽象概念，如“自由”“成长”“压力”。
- 高分表现：有具体物件、场景、动作、光线、情绪。

### 6. 旁白传播力，10 分

检查旁白是否适合短视频，不要太像议论文。

- 低分表现：解释性太强、节奏平。
- 高分表现：有停顿、有反转、有余味。

### 7. 发布标题，5 分

检查小红书/B站标题是否有点击欲。

- 低分表现：标题泛泛。
- 高分表现：短、尖、能引发讨论。

## 四、审稿输出格式

每次审稿必须输出：

```markdown
# 内容审稿报告

## 审稿文件

文件路径：

## 总分

xx / 100

## 是否建议发布

- 可直接发布
- 建议小修后发布
- 建议大修
- 不建议发布

## 分项评分

| 维度 | 分数 | 问题 | 修改建议 |
|---|---:|---|---|
| 命题锋利度 | /20 | | |
| 核心价值冲突 | /20 | | |
| A/B 两难张力 | /15 | | |
| 内容原创感 | /15 | | |
| 分镜画面感 | /15 | | |
| 旁白传播力 | /10 | | |
| 发布标题 | /5 | | |

## 最大问题

用 3 到 5 句话指出这篇内容最大的问题。不要只说“还可以”，必须具体。

## 最值得保留的部分

指出最有价值的 1 到 3 个片段。

## 必改建议

列出最优先修改的 3 条建议。

## 可选精修方向

如果要继续提高，给出 2 到 3 个方向。
```

## 五、审稿规则

1. 不要只夸。
2. 不要因为结构完整就判定优秀。
3. 必须指出模板味、空泛、主题漂移、标题平淡等问题。
4. 如果 A/B 选项一边明显更正确，必须指出。
5. 如果分镜不能被画出来，必须指出。
6. 如果旁白像作文结尾，必须指出。
7. 如果内容值得发布，也要说明为什么值得发布。

## 六、测试任务

创建完成后，用本标准审查：

`outputs/growth-rules-freedom.md`

只输出审稿报告，不修改原文。
