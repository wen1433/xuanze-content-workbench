# 《选择之间》命题调集系统 v0.1

这个文件用于指导 Codex 把普通议题种子转化成《选择之间》风格的两难命题。

## 使用流程

第一步：
把看到的热榜、评论区观点、社会现象、个人想法粘贴到 `topics/raw-seeds.txt`。

第二步：
让 Codex 读取 `topics/raw-seeds.txt`，并按照本文件规则生成 `topics/generated-topics.md`。

第三步：
用户从 `topics/generated-topics.md` 里挑选命题，放入 `topics/selected-topics.md`。

第四步：
使用 `.agents/skills/xuanze-between` skill 批量生成内容素材。

## 转化原则

- 不搬运原始标题。
- 不复制评论原文。
- 只提炼冲突，不复述热闹。
- 每条命题必须有 A/B 张力。
- 命题要能让用户做出选择，而不是只表达观点。
- 命题不要变成普通投票、情绪宣泄或立场口号。
- 命题最好能落入以下主题之一：
  - AI 技术
  - 死亡意义
  - 道德救助
  - 亲密关系
  - 自由安全
  - 成长未来
  - 记忆痛苦

## 转化步骤

1. 读取原始议题种子，判断它背后的真实问题。
2. 去掉具体平台、人物、热搜语气和情绪化表达。
3. 提炼两个互相拉扯的价值：
   - 例如：稳定回应 vs 真实他者
   - 例如：适应规则 vs 成为自己
   - 例如：普遍伦理 vs 特殊责任
4. 改写成一句可选择、可讨论的命题。
5. 标注识别主题和核心价值冲突。
6. 评估推荐等级：高 / 中 / 低。
7. 判断是否适合继续用 `xuanze-between` skill 生成完整内容素材。

## 命题格式示例

原始议题：
AI 陪伴软件爆火

转化命题：
如果 AI 比真实的人更稳定地陪伴你，你还需要真实关系吗？

核心冲突：
稳定回应 vs 真实他者 / 效率陪伴 vs 人类关系

## 质量检查

生成 `topics/generated-topics.md` 前，逐条检查：

- 是否只是换了一种说法复述原始标题？
- 是否真的形成了 A/B 两难，而不是单边批判？
- 是否能看出用户必须在两个价值之间选择？
- 是否避免了平台、热搜、人名等容易过期的信息？
- 是否能被 `xuanze-between` skill 继续扩写？

如果不通过，重写该条命题。
